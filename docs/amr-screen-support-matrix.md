# AMR Screen Support Matrix

This matrix maps the canonical Authentication Method Reference (AMR) vocabulary to the web and non-web screens required to safely produce, manage, inspect, or configure each AMR.

Status reflects end-to-end native support, not whether an AMR string passes vocabulary validation. A screen is included only when it is required for the AMR itself or for the authenticator evidence supporting it.

## Legend

Status:

- 🟢 Direct core support
- 🟡 Partial or indirect support
- 🔴 Native capability missing
- 🔵 Derived or contextual AMR

Screens:

- 📄 Existing screen
- 🛠 Existing screen requiring material extension
- 🆕 New screen required
- ♻ Shared screen or ceremony
- — No AMR-specific screen
- ⚠ Display only when supported by verified evidence

## Implementation note

The central implementation should remain one server-driven `CeremonyShell`. Login, enrollment, verification, step-up, recovery, replacement, linking, and reauthentication are ceremony purposes rather than separate authentication engines.
