# OIDC Party Vendor Support Matrix

Research date: 2026-05-27

This is a compact support matrix for OIDC/OAuth parties and subsystems. Each cell combines a support marker with the concrete package, API, SDK, UIX, hosted UI, or admin/control-plane surface that provides coverage.

## Legend

| Marker | Meaning |
|---|---|
| ✅ | Present first-class support in a documented product/API/package/UI surface. |
| 🟡 | Partial, adjacent, or supported through a broader/general surface rather than a dedicated product surface. |
| 🟣 | Present as a planned or newly framed `tigrbl_auth` product lane; implementation/product maturity still needs follow-through. |
| 🚧 | Underframed or incomplete in `tigrbl_auth`; primitives may exist but the product/API surface is not first-class. |
| ❌ | No clear documented first-class support in the researched vendor surface. |
